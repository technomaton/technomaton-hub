# Invisible Pain Discovery

Discover invisible pain points for the market or workflow specified in $ARGUMENTS.

Read `skills/pmf-assessment/AI_PMF_CORE.md` from this pack for the invisible pain methodology.

Apply the 4-step discovery process:

1. **Shadow workflows** — What do users actually do day-to-day? Map the real process, not the documented one.
2. **Time allocation** — Where does 40%+ of time go on tasks considered "just part of the job"?
3. **Workaround detection** — Identify elaborate manual processes that substitute for missing automation.
4. **Zero-creativity tasks** — Find high-volume actions requiring no human judgment.

For each discovered pain point, assess whether it is:
- **AI-shaped** — Involves ambiguity, reasoning, or judgment (good fit for AI)
- **Logic-shaped** — Deterministic rules, simple automation suffices (not AI-differentiated)

Output a ranked list of invisible pain points with AI-shape assessment and estimated opportunity size.

Default output language is Czech. If the user writes in English, respond in English.
